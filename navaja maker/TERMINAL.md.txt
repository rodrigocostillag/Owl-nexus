# TERMINAL — Sistema Unificado de Consolas

## 1. Propósito

El módulo TERMINAL permite:

- Ejecutar comandos locales (Linux)
- Conectarse a dispositivos remotos (SSH)
- Interactuar con múltiples dispositivos simultáneamente
- Servir como interfaz base para automatización y AI

---

## 2. Tipo de plugin

- Function Plugin (nivel raíz)
- Integrado con SSH, AI, Device Manager

---

## 3. Concepto clave

> Una sola terminal para controlar todo el sistema (local + remoto)

---

## 4. Tipos de terminal

---

### 4.1 Terminal local

- Ejecuta comandos en el sistema host
- Bash / Zsh

---

### 4.2 Terminal remota (SSH)

- Conexión a dispositivos
- Usa Credential Manager
- Basado en Device Manager

---

### 4.3 Terminal multi-dispositivo

- Ejecuta comandos en múltiples dispositivos
- Paralelo o secuencial

---

### 5.1 Ejecución de comandos

## 22. Sistema de Shells (Terminal real extensible)

### Propósito

Permitir que la terminal utilice shells reales del sistema (especialmente Linux) para aprovechar:

- Plugins existentes
- Temas
- Configuraciones avanzadas
- Ecosistema real (bash, zsh, etc.)

---

## 23. Tipos de shell soportados

---

### 23.1 Shell local del sistema

- Bash
- Zsh
- Fish

---

### 23.2 Shell embebido

- Ejecutado dentro de la aplicación
- Controlado por el sistema

---

### 23.3 Shell remoto (SSH)

- Replica el shell del dispositivo remoto
- Mantiene compatibilidad con entorno real

---

## 24. Configuración de shell

## 33. Objetivos prioritarios del módulo

### Prioridad 1 — Integración con terminales Linux reales

El sistema debe enfocarse primero en:

- Usar shells reales del sistema (bash, zsh, fish)
- Ejecutar entorno Linux completo
- Evitar reinventar funcionalidades existentes

---

### Requisitos clave

- Compatibilidad con Zsh como opción principal
- Soporte para frameworks existentes:
  - oh-my-zsh
  - starship
  - powerlevel10k

---

### Objetivo práctico

> Poder usar la terminal igual que en una distro Linux real

---

### Capacidades esperadas

- Soporte completo de comandos Linux
- Uso de herramientas estándar:
  - git
  - ssh
  - tmux
  - htop
- Configuración mediante:
  - .bashrc
  - .zshrc

---

### Personalización

- Temas avanzados
- Colores dinámicos
- Prompts inteligentes
- Alias personalizados

---

### Integración con el sistema

- Versionado de configuración (GIT)
- Uso por IA
- Sincronización entre dispositivos

---

### Resultado esperado

Un entorno donde:

- La terminal no se siente limitada
- Se comporta como un sistema Linux real
- Se pueden reutilizar herramientas existentes
