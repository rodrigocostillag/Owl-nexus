# AI — Sistema de Inteligencia Artificial y Automatización

## 1. Propósito

El módulo de IA permite:

- Interactuar con toda la aplicación mediante lenguaje natural
- Automatizar tareas complejas
- Controlar múltiples dispositivos
- Ejecutar acciones en otros plugins (Dashboard, Watcher, SSH, etc.)

La IA actúa como un **orquestador central del sistema**.

---

## 2. Tipo de plugin

- Function Plugin (nivel raíz)
- Visible en sidebar
- También accesible como overlay global

---

## 3. Comportamiento principal

### Activación

- Botón global ON/OFF
- Al activarse:
  - Se abre un panel lateral derecho (tipo overlay)
  - Estilo similar a asistentes tipo “side panel”

---

## 4. UI del Chat

### Panel lateral derecho

Contiene:

- Historial de conversación
- Input de comandos
- Estado del agente
- Indicador de tokens

---

### Características del chat

- Conversación continua
- Contexto persistente
- Capacidad de ejecutar acciones
- Streaming de respuestas

---

## 5. Capacidades de la IA

La IA puede:

- Leer estado del sistema
- Ejecutar comandos en dispositivos
- Modificar configuraciones
- Consultar datos (Watcher, Dashboard)
- Automatizar tareas multi-dispositivo

---

### Ejemplos de uso

- "Escanea la red y dime qué dispositivos nuevos hay"
- "Actualiza todos los ESP32"
- "Conéctate por SSH a la impresora"
- "Configura el rango de red a 192.168.0.1 - 192.168.1.255"

---

## 6. Integración con el sistema

La IA tiene acceso a:

- Dashboard
- Watcher
- SSH
- Device Manager
- Plugins activos

---

### Modelo de interacción


Usuario → IA
IA → interpreta intención
IA → genera plan
IA → ejecuta acciones
IA → devuelve resultado

### Propósito

Permitir múltiples agentes especializados.

---

### Ejemplos de agentes

- Usar un solo agente

### Ejemplo

→ Network Agent escanea
→ SSH Agent ejecuta comandos

## 19. Sistema de Tools (Herramientas de IA)

### Propósito

Permitir que la IA y los agentes puedan ejecutar acciones estructuradas dentro del sistema mediante herramientas (tools).

Las tools son la forma en que la IA interactúa realmente con:

- Plugins
- Dispositivos
- Servicios del sistema

---

## 20. Concepto de Tool

Una Tool es:

- Una función estructurada
- Con inputs definidos
- Con outputs claros
- Ejecutable por la IA

---

### Ejemplos de tools

- scan_network
- ssh_execute
- get_devices
- update_device
- open_web_interface

---

## 21. Estructura conceptual de una Tool

Cada tool define:

- Nombre
- Descripción
- Parámetros
- Permisos requeridos
- Plugin al que pertenece

---

### Ejemplo conceptual


Tool: scan_network

Inputs:

range
speed

Output:

lista de dispositivos

---

## 22. Registro de Tools

Las tools pueden registrarse desde:

- CORE
- Plugins
- Sub-plugins

---

### Importante

Esto permite que:

- Cada plugin exponga capacidades a la IA
- El sistema crezca dinámicamente
- La IA tenga acceso estructurado a nuevas funciones

---

## 23. Tools por Plugin

Cada plugin puede:

- Definir sus propias tools
- Extender tools existentes
- Restringir acceso

---

### Ejemplo

Plugin Watcher:

- scan_network
- get_network_info

Plugin SSH:

- execute_command
- upload_file
- restart_service

---

## 24. Tools para Agentes

Cada agente puede:

- Tener acceso a ciertas tools
- Tener restricciones
- Tener prioridad sobre otras tools

---

### Configuración de agentes con tools

- Lista de tools permitidas
- Nivel de acceso
- Contexto disponible

---

## 25. Sistema de permisos

Las tools deben poder tener:

- Permisos básicos
- Permisos críticos (requieren confirmación)
- Acceso restringido por agente

---

## 26. Ejecución de Tools

Flujo:


IA recibe prompt
→ Analiza intención
→ Selecciona tool
→ Ejecuta tool
→ Recibe resultado
→ Responde al usuario


---

## 27. Tools dinámicas

El sistema permite:

- Crear nuevas tools en runtime
- Modificar tools existentes
- Activar/desactivar tools

---

## 28. Integración con Plugins (Servitización)

Las tools funcionan como capa de servicios:

- Cada plugin expone funcionalidades como tools
- La IA consume esas tools como servicios

---

### Concepto clave

> Plugin = Feature  
> Tool = API del plugin para la IA

---

## 29. Extensión del sistema

Esto permite:

- Añadir nuevas capacidades sin modificar el core
- Integrar nuevos plugins fácilmente
- Escalar el sistema con IA

---

## 30. Ejemplo completo


Prompt:
"Escanea la red y reinicia todos los dispositivos offline"

IA:
→ usa scan_network (Watcher)
→ filtra dispositivos offline
→ usa execute_command (SSH)
→ ejecuta reboot


---

## 31. Objetivo final

Crear un sistema donde:

- La IA no ejecuta código directo
- La IA usa tools estructuradas
- Todo el sistema es controlable de forma segura y modular
